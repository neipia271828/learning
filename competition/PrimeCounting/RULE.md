# 素数の数を数える

## 課題

整数 `N` が与えられたとき、`1` 以上 `N` 以下に含まれる素数の個数を求めてください。

例:

- `N = 10` のとき、素数は `2, 3, 5, 7` の 4 個なので、答えは `4`
- `N = 100` のとき、答えは `25`

## この課題の目的

この課題の目的は、素数に関する数学的な性質と、それをプログラムとしてどう実装するかの理解を深めることです。

そのため、単に速いコードを書くことだけでなく、なぜその方法で正しく数えられるのか、なぜ速くなるのかを説明できることを重視します。

## 素数とは

`1` より大きい整数のうち、`1` とその数自身以外では割り切れない数を素数とします。

## 入力

標準入力から整数 `N` を 1 つ受け取ること。

```text
N
```

## 出力

`1` 以上 `N` 以下の素数の個数を、標準出力に 1 行で出力すること。

```text
count
```

## 制約

- `1 <= N <= 1000000000`
- 使用言語は自由
- 使用する数学的な考え方、アルゴリズム、データ構造、実装上の工夫は、すべて自分の言葉で説明できること
- AI、書籍、Web サイト等を参考にしてもよいが、提出時に「なぜそれで正しいか」「なぜ速いか」を説明できないものは使わないこと

## ライブラリ利用

- 標準ライブラリの使用は可
- 入出力、配列、ビット演算、基本的なコレクション、時刻計測などの補助的なライブラリは使用可
- ただし、素数判定、素数列挙、素数計数、篩の実装など、この課題の本質部分をライブラリに任せることは禁止
- たとえば「与えた `N` 以下の素数を返す」「素数の個数を返す」「高速な素数判定を内部で行う」機能をそのまま呼び出す使い方は禁止
- 自作コードが主役であり、ライブラリは補助にとどまること
- 使用したライブラリについては、何のために使ったかを説明できること
- 主催者が「この課題の理解を深める目的に反する」と判断した場合、そのライブラリ利用は認めないことがある

## 評価

- まず正しい答えを返すことを最優先とする
- そのうえで、実行速度を絶対評価で判定する
- 実行速度の計測は、主催者が用意した同じ実行環境で行う
- ベンチマーク入力は `N = 1000000000` とする
- 計測は 3 回行い、その中央値を正式記録とする
- 中央値が `3.00` 秒以下なら景品
- 参考として、`10.00` 秒以下なら十分速い実装とみなす

## 提出物

以下を提出すること。

- ソースコード
- 実行方法
- 次の説明
  - どんな方針で解いたか
  - 計算量がどのくらいか
  - なぜその方法で正しいといえるか
  - どこを高速化したか

## 説明で確認したいこと

提出後、必要に応じて以下を口頭または文章で確認します。

- 素数をどう判定しているか
- すべての数を毎回割る方法と比べて、何が改善されているか
- 自分の実装の計算量はどこで決まるか
- `N` が大きくなったとき、どこがボトルネックになるか

## サンプル

### 入力例 1

```text
10
```

### 出力例 1

```text
4
```

### 入力例 2

```text
100
```

### 出力例 2

```text
25
```

## ヒント

- まずは正しく動く単純な方法で作ってよい
- その後、より速い方法に改善するとよい
- 「速い方法を使った」だけでは不十分で、「なぜその方法でよいのか」を説明できることが大事
